TrueNAS Server Build

Custom-built homelab server running TrueNAS SCALE with comprehensive media automation, Bitcoin full node, photo management, and home automation integrations.

Project Overview

This is the complete hardware build and infrastructure setup for my TrueNAS server. Built from the ground up with carefully selected components for reliability, performance, and expandability. Overcame initial hardware challenges (defective motherboard RMA) to create a rock-solid foundation for media automation, Bitcoin operations, photo storage, and home automation backup infrastructure.

Hardware Specifications

CPU Intel Core i5-12500 (6 cores, 12 threads, 3.0-4.6GHz)
RAM 32GB DDR4
Motherboard ASRock B760M Pro RS (replacement after defective unit RMA)
Storage Pool 2x 8TB Seagate IronWolf (ZFS Mirror RAID1)
Usable Capacity 7.18TB (14.5TB raw, mirrored)
Operating System TrueNAS SCALE 25.10.0

Build Challenge: First motherboard was DOA (wouldn't POST). Successfully RMA'd and replaced, then completed installation without further issues.

Zero-trust VPN with subnet routing enabled for entire network access. Allows remote BitAxe access via secure mesh networking.

Backup access method with host network mode. Routes configured for Overseerr, Radarr, Sonarr, Transmission, Plex, and Immich.

Plex Direct Access

Port 32400 (TCP) forwarded on router for direct connections, eliminating relay lag during streaming.

Synology to TrueNAS Migration

Complete data migration from Synology DS124 using rsync with auto-retry logic to handle connection interruptions.

Source Synology DS124 (/mnt/synology_temp)
Total Data ~2.2TB
Method rsync with auto-retry loop

Migration Data Breakdown

  • Movies: 75GB (including extras)

Auto-Retry Solution: Used while loop with rsync to automatically resume transfers after connection suspensions. Preserved Transmission seeding sessions throughout migration.

Security & Permissions

Media User UID 3000, GID 3000
Standard Permissions 775 (rwxrwxr-x)
Transmission User UID 3000, GID 3000
All *arr Apps UID 3000, GID 3000

Network Security

  • Tailscale: Zero-trust mesh VPN for all remote access
  • Cloudflare Tunnel: Encrypted tunnel without port forwarding (except Plex)
  • TrueNAS Firewall: Default restrictive configuration
  • VPN for Torrenting: All download traffic through Mullvad WireGuard
  • Authentication: All exposed services password-protected or using Plex SSO

Troubleshooting & Solutions

Common issues and their solutions
Issue Problem Solution
Permission Errors Radarr couldn't move files from incomplete to complete Recursively set ownership to 3000:3000 and permissions to 775 on downloads directory
Samba Backup "Cannot access share" errors SMB service wasn't running; enabled and set to auto-start
Cloudflared Crashes Container starts then immediately stops Enable "Host Network" mode in app configuration
File Browser HTTPS "Client sent HTTP request to HTTPS server" Changed Cloudflare route to https:// (ultimately removed from tunnel)
Indexer API Limits Connection refused errors from excessive API calls Wait 24 hours during ban period; reduce Prowlarr sync frequency
